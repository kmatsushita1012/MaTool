# タブ構成移行計画書

現在はホーム画面を起点に各の画面に遷移する構成だがこれをタブ構成に変更する。

## タブ構成
- 地図タブ
- 紹介タブ
- お知らせタブ
- 設定/管理タブ

備考
- お知らせ機能は未実装なので将来的に拡張
- その他タブに相当する画面はすでに存在する

## 各タブ詳細設計

###　地図タブ
以下の階層で構成する
- 祭典選択画面
- 祭典個別画面

#### 祭典選択画面
Festival.baseの座標にピンを立てる
ピンをタップすると祭典別画面に遷移する

#### 祭典個別画面
現在のPublicMapFeatureに相当
PickerのデザインはiOS標準に変更

町Picker
- 規制区域＆出店
- 現在地一覧
- 各町詳細

日程Picker

###　紹介タブ
以下の階層で構成する
- 祭典選択画面
- 祭典個別画面
- 町個別画面

#### 祭典選択画面
各祭典の基本情報をリスト形式で掲載
- 祭典名
- 画像
- 都道府県
- 市区町村

#### 祭典個別画面
SegmentedControlで2つのサブタブに分割
- 祭典の詳細情報
- 各町のリスト

詳細情報にはFestivalに含まれるその他情報と地図を掲載(FestivalInfoに相当)
各町リストには拡張の以下の項目を掲載
- 町名
- グループ
- 画像

#### 町個別画面
DistrictInfoに相当

###　設定/管理タブ

設定/管理タブはログイン状態で変化
- 未ログインなら各種設定項目+ログインボタン
- ログイン済なら管理者ダッシュボード+右上に設定画面への遷移ボタン
- 編集画面はフルスクリーン遷移でタブバーを排除する

設定項目は以下
- デフォルトの祭(ログイン中は設定不可)
- デフォルトの町(ログイン中は設定不可)
- 遷移パターン(後述)
- その他(今後の要件による)

ログイン時には管理者の権限に応じてデフォルトの祭/町が決まる
- 本部は所属する祭&町は未設定
- 各町管理者は所属する祭&所属する町


## タブ間の遷移パターン

- 追従モード　(デフォルト)
- 非追従モード

### 追従モード
地図タブと紹介タブで対応する画面階層に遷移する

| 地図タブ | 紹介タブ |
| --- | --- |
| 祭典選択画面 | 祭典選択画面 |
| 祭典個別画面(現在地一覧) | 祭典個別画面 |
| 祭典個別画面(各町) | 町個別画面 |

### 非追従モード
各タブは独立して動作する
